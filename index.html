Design Thoughts

How I think about building things.

Design

Design thinking made me a better engineer.

I didn't start with code — I started with how things look and feel. That sequencing still shapes how I build: UX considerations inform every architectural decision I make. The best engineering decisions are almost always also the best design decisions.

AI

LLMs are infrastructure, not magic.

The hard part of building with LLMs isn't the API call — it's structured output, failure handling, and designing systems that behave predictably when the model doesn't. Responsible AI output design is non-negotiable when users make real decisions from model responses.

Building

Ship for the failure case, not the happy path.

Every meaningful project I've built — a women's safety app, a legal AI for rural India — forced me to think about what happens when things go wrong. Safety-critical product design starts with the failure scenario, not the happy path.
